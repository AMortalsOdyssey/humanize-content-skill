# Style Rule: blog-of-anthropic

Use this rule for content that should feel like a grounded product, AI, engineering, collaboration, or organizational essay inspired by Anthropic's official blog style. Do not imitate exact wording. Extract the writing mechanism: sober framing, concrete work examples, clear boundaries, and evidence-rich advice.

## Shape

Start with a real shift:

- Bad: "AI agents are reshaping the future of work."
- Better: "Work with agents used to mean one person and one chat window. The harder problem starts when several people and several agents need to share the same goal."

Move in this order:

1. Previous way of working.
2. New pressure or failure mode.
3. The operating mechanism that fixes it.
4. Concrete roles, artifacts, permissions, tools, or workflows.
5. Verification, checklist, or next action.

## Sentence-Level Rules

- Use calm confidence. Avoid sales urgency.
- Prefer "what changed" over "why this is revolutionary".
- Define abstract concepts through objects people can inspect: files, channels, roles, credentials, screenshots, tests, commands, logs, schemas, meeting notes.
- Let examples carry emotion. Do not over-explain the moral.
- Use first person only when the speaker has actually lived the example.
- End with a useful question, checklist, or action, not a grand conclusion.

## Evidence Density

Each major point should include at least one of:

- A before/after workflow.
- A named actor or role.
- A concrete artifact.
- A failure mode.
- A validation method.
- A boundary or caveat.

If a paragraph has none of these, rewrite it or delete it.

## Common Moves

Use these moves to replace AI-like polish:

- "This matters because..." -> "The failure shows up when..."
- "It improves efficiency..." -> "It removes one handoff:..."
- "Teams can collaborate better..." -> "The whole team can now see the same artifact..."
- "Best practice..." -> "A useful rule is..."
- "Future outlook..." -> "Questions to ask before adopting it..."

## Avoid

- "赋能", "重塑", "革命性", "范式转移", "极致体验", "无限可能".
- Three abstract benefits in a row.
- A paragraph that only praises the idea.
- "不仅仅是 X, 更是 Y" as the main rhetorical engine.
- Ending with "未来已来" or any equivalent slogan.

## Good Ending Patterns

- "Before you add more agents, ask whether they share the same source of truth."
- "The test is simple: can another person open the files and continue the work?"
- "Start with one workflow, one owner, and one proof of completion."
- "If the answer cannot be verified, it is not done yet."

# Medium Rules: Video

Use for short videos, explainers, demos, voiceovers, talking-head scripts, and storyboard outlines.

## Principle

Video should show evidence before it explains the lesson. Replace abstract narration with visible moments: screen recordings, failed outputs, timelines, before/after artifacts, checklists, or the speaker's real working environment.

## Short Video Structure

1. Cold open: a visible failure, surprising artifact, or direct claim.
2. Setup: who was trying to do what.
3. Demonstration: the workflow or mistake on screen.
4. Mechanism: the rule that explains the result.
5. Proof: command output, screenshot, metric, side-by-side comparison, or recovered artifact.
6. Close: one practical rule or question.

## Script Rules

- Write for speaking, not reading. Use shorter clauses.
- Avoid "今天给大家分享", "干货满满", and "一定要看到最后" unless the creator's own voice uses them ironically.
- Add shot notes when the image matters: `[screen: failed merge conflict]`, `[close-up: requirement matrix]`.
- Give the editor objects to cut to, not generic concepts.
- If there is a claim, add the shot that proves it.

## Human Texture

Use small, specific admissions:

- "这里我第一次做错了。"
- "看起来很慢，但后面省掉的是返工。"
- "我当时以为是 Agent 不够强，后来发现是事实源乱了。"

Do not over-script emotion. Let the visible sequence create trust.
